Задача 1: Банковская система с аналитикой
Описание:
Разработайте систему для банка, включающую:
    • Разные типы счетов: ТекущийСчет, СберегательныйСчет, КредитныйСчет
    • Клиентов, которые могут владеть несколькими счетами
    • Аналитический модуль для расчета статистики по счетам
Требуемые классы:
    1. БанковскийСчет (базовый абстрактный класс):
        ◦ Атрибуты: номер_счета, баланс, владелец, история_операций (список)
        ◦ Методы:
            ▪ пополнить(сумма): добавляет сумму к балансу, записывает операцию в историю
            ▪ снять(сумма): проверяет достаточно ли средств (для кредитного счета разрешен овердрафт), записывает операцию
            ▪ @abstractmethod получить_лимит_снятия(): возвращает максимальную сумму для снятия
            ▪ @classmethod создать_счет(cls, владелец, начальный_баланс=0): создает экземпляр счета
    2. ТекущийСчет (наследуется от БанковскийСчет):
        ◦ Лимит снятия: 50 000 в день
        ◦ Дополнительно: список привязанных карт (агрегация)
    3. СберегательныйСчет (наследуется от БанковскийСчет):
        ◦ Лимит снятия: 100 000 в месяц
        ◦ Метод начислить_проценты(ставка)
    4. КредитныйСчет (наследуется от БанковскийСчет):
        ◦ Лимит снятия: кредитный лимит + текущий баланс
        ◦ Атрибут кредитный_лимит
    5. Клиент (агрегация счетов):
        ◦ Атрибуты: имя, счета (словарь: тип счета → список счетов)
        ◦ Методы:
            ▪ открыть_счет(тип_счета, **kwargs)
            ▪ получить_общий_баланс(): сумма балансов всех счетов
            ▪ найти_счета_по_типу(тип)
    6. БанкАналитика (статические методы):
        ◦ @staticmethod рассчитать_медианный_баланс(список_счетов)
        ◦ @staticmethod найти_крупные_транзакции(счет, порог)
        ◦ @classmethod сгенерировать_отчет(cls, клиент): возвращает PDF/HTML отчет с графиками
Что должны реализовать студенты:
    • Композицию: Счета создаются и уничтожаются вместе с клиентом
    • Агрегацию: Карты могут существовать без текущего счета
    • Полиморфизм: Разную реализацию получить_лимит_снятия()
    • Статические методы для математических расчетов
    • Методы класса для альтернативных конструкторов


Задача 2: Игровая система с инвентарем
Описание:
Создайте RPG-систему с:
    • Персонажами разных классов (воин, маг, лучник)
    • Инвентарем с ограниченным весом/ячейками
    • Экипировкой предметов
    • Системой крафта
Требуемые классы:
    1. Персонаж (базовый класс):
        ◦ Атрибуты: имя, здоровье, инвентарь (композиция), экипировка (слоты: оружие, броня)
        ◦ Методы:
            ▪ поднять_предмет(предмет): проверяет свободное место
            ▪ использовать_предмет(название)
            ▪ @classmethod создать_персонажа(cls, класс, имя)
    2. Воин, Маг, Лучник (наследуются от Персонаж):
        ◦ Ограничения по экипировке (воины не могут использовать посохи)
        ◦ Уникальные бонусы к предметам
    3. Инвентарь (композиция):
        ◦ Атрибуты: макс_вес, предметы (словарь: предмет → количество)
        ◦ Методы:
            ▪ добавить_предмет(предмет)
            ▪ удалить_предмет(предмет)
            ▪ @staticmethod рассчитать_вес(предметы)
            ▪ @classmethod создать_стартовый_инвентарь(cls): вода, хлеб, 50 золота
    4. Предмет (базовый класс):
        ◦ Атрибуты: название, вес, стоимость, тип (оружие/броня/расходник)
        ◦ Метод использовать(персонаж)
    5. Оружие, Броня, Зелье (наследуются от Предмет):
        ◦ Уникальные атрибуты: урон, защита, эффект
        ◦ Переопределение метода использовать()
    6. КрафтСистема (статичные методы):
        ◦ @staticmethod доступные_рецепты(уровень_персонажа)
        ◦ @classmethod создать_предмет(cls, рецепт, ингредиенты): проверяет наличие ингредиентов
        ◦ @classmethod разобрать_предмет(cls, предмет): возвращает материалы
Что должны реализовать студенты:
    • Композицию: Инвентарь не существует без персонажа
    • Агрегацию: Предметы могут существовать вне инвентаря
    • Наследование для разных типов предметов
    • Статические методы для расчетов веса/стоимости
    • Методы класса для фабричного создания объектов
    • Полиморфизм использования предметов
